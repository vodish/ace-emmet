### id для контента

this.$textLayer.$lines.cells
содержит массив ячеек cell для хранения текстовых блоков, которые рендерятся редактором

- реализуй стор в рамках ace.js который будет выдавать числовые уникальные id для cell.element.textContent
- стор должен перемещать id текста вслед за номером ячейки cell.row
- я хочу отслеживать какая именно строка передвинулась в какую ячейку
- новые строки должны регистрироваться в хранилище, получая новый id
- при загрузке страницы index.html я сделал отладочную инфо по редактору в диве #ids
id=undefined row=0 "two"
id=undefined row=1 "one"
id=undefined row=2 ""

я ожидаю что перемещение первой строки в редакторе приведет к изменению в #ids, который изменится вот так
id=1 row=0 "two"
id=0 row=1 "one"
id=3 row=2 ""

- ID присваивается тексту строки и следует за ним при перемещении
- При изменении текста в строке без перемещения — ID остаётся прежним
- Только НОВЫЕ строки получают новые уникальные ID (через инкрементальный счётчик nextId)
- ID никогда не должен повторятся

- для новых пустых строк должны создаваться новые ID. ID не могут повторятся!
- id не должен менятся если текст остался в тойже ячейке cell.row


-
this.cells — массив объектов ячеек (cell), каждый элемент имеет следующие свойства:
    
    element: DOM-элемент div, которому присваивается класс ace_line (или ace_line_group).
    
    text: строка, содержащая текстовое содержимое строки (изначально пустая, заполняется при рендеринге).
    
    row: номер строки в документе (индекс).
